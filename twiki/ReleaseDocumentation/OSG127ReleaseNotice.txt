<div class="announcement-head" style="border: thin solid #000; background-color: #F90;">
  <h1 align="center">OSG 1.2.7 Update Notification</h1>
</div>
<br />

<div class="date" style="font-family: Tahoma, Geneva, sans-serif; font-size: 14px; font-weight: bold; border: thin solid #000; background-color: #F93;">Date: <span class="date-text" style="font-size: 14px; font-weight: normal;">March 8, 2010</span></div>
<br />
<div class="update-heading" style="font-family: Tahoma, Geneva, sans-serif; font-size: 14px; font-weight: bold; background-color: #369; border: thin solid #000; color: #CCC;">Affected Components</div>
<div>This update affects all OSG installations.</div>
<br />
<div class="update-heading" style="font-family: Tahoma, Geneva, sans-serif; font-size: 14px; font-weight: bold; background-color: #369; border: thin solid #000; color: #CCC;">Summary</div>
<div> This is a medium priority security update that corrects issues in several components.  The fixes include updates to several components to properly handle null bytes in a DN handle and a fix for a DOS attack on squid. <br />
</div>
<div class="update-heading" style="font-family: Tahoma, Geneva, sans-serif; font-size: 14px; font-weight: bold; background-color: #369; border: thin solid #000; color: #CCC;">Components updated</div>

<div>
  <p>This release also updates several software components, see the complete list below.</p>
  <ul>
    <li>wget</li>
    <li>osg-config</li>
    <li>osg-version</li>
    <li>Java 1.5</li>
    <li>Java 1.6</li>
    <li>OpenLDAP </li>
    <li>GFAL and lcg-utils (no version change, but it was rebuilt to match the security update for OpenLDAP)</li>
  </ul>
</div>
<br />
<div class="update-heading" style="font-family: Tahoma, Geneva, sans-serif; font-size: 14px; font-weight: bold; background-color: #369; border: thin solid #000; color: #CCC;">Issues Fixed</div>
<div>
  The components that have been updated may also include bugfixes. Please see the <a href="http://vdt.cs.wisc.edu/releases/2.0.0/release-p14.html">VDT release notes</a> for more details.</div>

<br />
