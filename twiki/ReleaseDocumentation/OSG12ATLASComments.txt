---++ Introduction
Testing for the OSG 1.2 release on the ITB 1.1.4 testbed.

---++ Setup
   * Using UC_ITB site since it has been configured in Panda, and has a nearby storage element with ATLAS data.
   * First problem: Panda queue is not configured correctly - email to experts. Resolved.
   * Autopilot status submission: http://panda.cern.ch:25980/server/pandamon/query?tp=pilots&accepts=UCITB_EDGE7  (UCITB_EDGE7 is the Panda queue name)
   * Queue information http://panda.cern.ch:25880/server/pandamon/query?tp=queue&id=UCITB_EDGE7-pbs
   
---++ Local Site Mover
For ATLAS purposes, UC_ITB is attached to the MWT2_UC.  It shares the (space-tokenized) storage elements of MWT2_UC, in particular ATLASPRODISK.  It also uses the installed ATLAS release (exported to UC_ITB nodes over NFS).  However, it utilizes the UC_ITB gatekeeper, job manager, and GUMS server.

The MWT2 storage element is dCache-based, and access to it is via a special set of "local site mover" tools invoked by ATLAS pilot jobs.  A custom version of these tools was developed for UC_ITB.   Additional sources of debug info: /var/log/lsm/* on the uct3 WNs. For reference, the lsm scripts are at: http://repo.mwt2.org/viewvc/lsm/UCITB/.
